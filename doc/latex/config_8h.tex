\doxysubsubsection{Подробное описание}
xmlrpc\+\_\+config.\+h is generated from xmlrpc\+\_\+config.\+h.\+in by \textquotesingle{}configure\textquotesingle{}.

The purpose of this file is to define stuff particular to the build environment being used to build Xmlrpc-\/c. Xmlrpc-\/c source files can \#include this file and have build-\/environment-\/independent source code.

A major goal of this file is to reduce conditional compilation in the other source files as much as possible. Even more, we want to avoid having to generate source code particular to a build environment except in this file. ~\newline


This file is N\+OT meant to be used by any code outside of the Xmlrpc-\/c source tree. There is a similar file that gets installed as $<$xmlrpc-\/c/config.\+h$>$ that performs the same function for Xmlrpc-\/c interface header files that get compiled as part of a user\textquotesingle{}s program.

This file just uses plain A\+C\+\_\+\+S\+U\+B\+ST substitution, the same as config.\+mk. Wherever you see @\+X\+XX@, that gets replaced by the value of \textquotesingle{}configure\textquotesingle{} variable X\+XX.

Logical macros are 0 or 1 instead of the more traditional defined and undefined. That\textquotesingle{}s so we can distinguish when compiling code between \char`\"{}false\char`\"{} and some problem with the code.
